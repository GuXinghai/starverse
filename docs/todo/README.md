# Todo 目录
本目录路径为 `/docs/todo`，用于记录待办事项（Backlog→Done）且不承载设计决策正文。

## 用途
- 统一存放待办事项文档，便于长期维护与追踪
- 与功能文档、决策记录分离，降低检索成本

## 命名规范
- 使用 `kebab-case` 命名
- 文件名聚焦能力或模块，例如 `restart-app.md`
- 若需区分阶段，可追加后缀，例如 `restart-app-mvp.md`

## 状态字段
- **Backlog**：已记录但尚未排期
- **Planned**：已排期，等待进入实施
- **In Progress**：实施中
- **Done**：已完成并关闭
- **Dropped**：不再执行或已被替代
