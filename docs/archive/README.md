## 🔍 如何使用归档

### 查找历史信息
1. 根据问题类型选择对应目录（重构/修复/优化/分析）
2. 查看目录下的 README.md 获取文档列表
3. 阅读具体文档了解历史实现细节

### 参考历史方案
归档文档可作为：
- 技术决策的历史依据
- 类似问题的解决方案参考
- 代码演进的追溯路径
- 新团队成员的学习资料

### 搜索技巧
```bash
# 在归档中搜索关键词
cd docs/archive
grep -r "关键词" .

# 查找特定文件
find . -name "*关键词*.md"
```

## 📋 归档原则

### 归档触发条件
文档满足以下任一条件即归档：
- ✅ 标题含 "COMPLETE"、"已完成"
- ✅ 完成日期距今 > 30 天
- ✅ 内容标注 "状态：已完成"
- ✅ 问题已解决且验证稳定
- ✅ 分析已完成并产生后续行动

### 不归档的文档
- ❌ 核心架构文档（长期有效）
- ❌ 开发指南和规范
- ❌ 当前活跃的实现计划
- ❌ 待办事项和正在进行的工作
